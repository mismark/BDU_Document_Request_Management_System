# BDU-GDRMS: Graduate Document Request & Management System

## 1. Introduction

The Bahir Dar University Graduate Document Request Management System (BDU-GDRMS) is a web-based platform designed to digitize and streamline the process of requesting, processing, and authenticating academic documents. It serves graduates, the university registrar, and external employers/verifiers.

## 2. System Architecture

## 3. Database Schema

The system relies on a relational database comprising four key tables:

1. **users**: Stores authentication details.
   - Roles: `graduate`, `registrar`, `admin`
2. **requests**: Tracks the lifecycle of a document request.
   - Statuses: `pending`, `approved`, `rejected`
3. **documents**: Stores authenticated document records.
   - Links strict `verification_id` to a request.
4. **verification_logs**: (Optional) Audit trail for document checks.

## 4. User Roles & Features

### 4.1 Graduate (Student)

- **Registration/Login:** Secure account creation.
- **Request Documents:** Submit requests for Transcripts, Degrees, etc.
- **Track Status:** Real-time visibility into "Pending" vs "Approved" status.
- **Digital Retrieval:** View/Download authenticated digital credentials.

### 4.2 Registrar (Staff)

- **Dashboard:** View a prioritized queue of pending requests.
- **Authentication:** Verify student details against internal records.
- **Issue/Reject:**
  - _Approve:_ Generates a unique, tamper-proof Verification ID.
  - _Reject:_ specific feedback on why a request was denied.

### 4.3 Administrator (Admin)

- **User Management:** Promote/Demote user roles (e.g., assigning new Registrars).
- **System Health:** Monitor system statistics (Total Users, Request Load).

### 4.4 Public (Employer/Verifier)

- **Public Verification:** Anyone with a valid Document ID can verify its authenticity without logging in.
- **Zero-Trust Validation:** The system queries the live database to confirm the document exists.

## 5. Security Features

- **Role-Based Access Control (RBAC):** Middleware checks prevent unauthorized access to sensitive dashboards.
- **Prepared Statements:** All SQL queries use prepared statements to prevent SQL Injection.
- **XSS Protection:** Input sanitization handles user-submitted text.
- **Password Hashing:** Uses PHP `password_hash()` (Bcrypt) for secure storage.

## 6. Installation & Setup

1. **Database:** Import `database.sql` into your MySQL server.
2. **Config:** Update `core/app.php` with your database credentials.
3. **Seeding:** Run `php seed_data.php` to generate initial Admin and Registrar accounts.
4. **Run:** Host on a local server (e.g., `localhost/bdu_gdrms`).
